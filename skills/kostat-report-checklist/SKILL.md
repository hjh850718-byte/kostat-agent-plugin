# kostat-report-checklist

## 트리거
- 사용자가 `/kostat-report` 입력 시 **즉시 실행**
- "보고서 써줘", "보고 작성", "이사님 보고" 등 보고서 관련 발화 시 실행 제안
- 보고서 작성 시작 전 반드시 0단계부터 순서대로 점검

## 작업 흐름

### 체크리스트 출력 및 인터랙티브 점검

사용자가 체크리스트를 요청하면 아래 4단계 + 사전 점검을 **단계별로 확인**하며 진행한다.
각 단계 완료 후 다음 단계로 넘어가기 전에 사용자 확인을 받는다.

---

## 체크리스트 본문

*(37signals 의사결정 원칙 → 배경/현황/옵션/권고안 프레임 적용)*

---

### 0단계. 보고 착수 전 — "이거 정말 보고가 필요한가?"

- [ ] 이 사안, 보고 없이 그냥 처리하면 무슨 일이 생기나? (실무 처리 가능 범위인지 먼저 점검)
- [ ] 이사/대표 결정이 꼭 필요한 사안인가, 아니면 내가 판단해서 통보만 하면 되는 사안인가?
- [ ] 일회성 이슈인가, 반복되는 패턴인가? (반복이면 → 보고서 대신 SOP/스킬화 검토)
- [ ] 마감 시한은 언제인가? (고객 대응 타이밍, 입찰/가격 회신 데드라인 등)

---

### 1단계. 배경(背景) 작성 시 — "왜 지금 이 문제가 생겼나"

- [ ] 이전에는 왜 이 결정이 안 내려졌는지 / 왜 미뤄졌는지 한 줄 정리
- [ ] 이 사안이 외부(고객/거래선) 발생인지, 내부(KOSTAT) 발생인지 구분
- [ ] 과거 유사 사례가 있었나? 있었다면 그때 결과는 어땠나 (회고 인용)

---

### 2단계. 현황(現況) 정리 시 — "데이터 vs 직감 구분"

- [ ] 지금 갖고 있는 데이터로 충분한가, 빠진 정보가 있나?
- [ ] 내 첫 직감은 무엇이었나 — 지금 데이터로 그 직감을 정당화하고 있는 건 아닌가? (확인편향 점검)
- [ ] 고객 관점 영향 vs KOSTAT 내부 영향, 양쪽을 분리해서 정리했나?

---

### 3단계. 옵션(Option) 설계 시 — "최소 2~3개, 비교 가능하게"

- [ ] 옵션을 더 작게 쪼갤 수 있는가? (하나의 큰 결정 → 여러 개의 작은 결정으로)
- [ ] 각 옵션은 **되돌리기 쉬운가(가역적)**, 아니면 한번 가면 못 돌아오나? → 표에 가역성 표시
- [ ] 각 옵션이 다른 결정(타 고객/타 부서/타 PO)에 영향을 주는가?
- [ ] 각 옵션이 누구의 업무량을 늘리나/줄이나? (영업팀, 생산, 물류 등)
- [ ] "고려 대상에서 일부를 빼면 더 쉬워지는 옵션"이 있는가? (단순화 가능성)

---

### 4단계. 권고안(권고) 작성 시 — "최종 점검"

- [ ] 1년 후 이 결정을 돌아봤을 때 어떻게 느껴질지 한 번 시뮬레이션
- [ ] 이 결정이 맞았는지/의미가 있었는지 **언제, 무엇으로** 확인할 수 있나? (후속 지표 설정)
- [ ] 결과가 눈에 바로 보이는 사안인가, 추적이 필요한 사안인가?
- [ ] 노력 대비 보상(ROI)이 보고할 만한 가치가 있는 사안인가?
- [ ] 결국 이게 돈(비용/매출/마진) 문제인가? → 권고안 첫 줄에 숫자로 명시

---

### 보고 직전 마지막 1문장 점검

> "이사님/대표님이 이 보고서만 보고도, 결정에 필요한 옵션·근거·리스크를 바로 파악할 수 있는가?"

---

## 출력 형식

체크리스트 점검 완료 후 아래 요약 출력:

```
📋 보고서 작성 체크리스트 점검 완료 — YYYY-MM-DD

0단계 (착수 전)   : ✅ / ⚠️ 미흡 N건
1단계 (배경)      : ✅ / ⚠️ 미흡 N건
2단계 (현황)      : ✅ / ⚠️ 미흡 N건
3단계 (옵션)      : ✅ / ⚠️ 미흡 N건
4단계 (권고안)    : ✅ / ⚠️ 미흡 N건

⚠️ 미흡 항목:
- [단계] [항목 내용]

최종 점검: "이사님/대표님이 이 보고서만 보고도, 결정에 필요한 옵션·근거·리스크를 바로 파악할 수 있는가?"
→ [사용자 답변 기재]
```

## 주의사항
- 0단계에서 보고가 불필요하다고 판단되면 **"실무 처리 가능 — 보고 생략 권장"** 메시지 출력 후 종료
- 반복 패턴 감지 시 → `/kostat-tal` 또는 SOP 문서화 제안
- 체크리스트는 보고서 **작성 전**에 반드시 실행 (작성 후 소급 점검은 효과 반감)
