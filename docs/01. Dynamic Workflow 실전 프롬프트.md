## 1. OOR (Open Order Report) — Fan-out 패턴

```
ultracode

OOR Excel 파일의 Bring Forward 항목 61개를 처리해야 한다.
Fan-out 워크플로우를 구성해서:
1. 분류 에이전트: 61개 항목을 [긴급/일반/보류] 3그룹으로 분류
2. 그룹별 서브에이전트 병렬 실행: 각 항목의 PO#, 납기, Status 교차 검증
3. 불일치 PO# 항목은 별도 플래그 처리
4. 합성 에이전트: 전체 결과 취합 → 컬러코딩 완료된 xlsx 출력

토큰 예산: 15k 이내
대상 파일: C:\Users\USER\Desktop\77. CLOUDE 정리용\
```

## 2. 커미션 인보이스 검증 — Adversarial Verification 패턴

```
ultracode

커미션 인보이스 생성 후 교차 검증 워크플로우:
1. 생성 에이전트: RAW DATA (US/ATP/MICRON 시트) 읽어서 요율 역산, 인보이스 초안 생성
2. 검증 에이전트(독립 컨텍스트): 생성 에이전트의 계산을 처음부터 재계산
   - 합계 금액 일치 여부
   - 요율 역산 정확도
   - SUMMARY 시트 수치 검증
3. 불일치 발견 시 자동 플래그 → 나에게 확인 요청
4. 검증 통과 시에만 최종 파일 저장

자기 확증 편향 방지: 생성/검증 에이전트는 서로 결과를 공유하지 않고 독립 실행
```

## 4. PO 일괄 처리 — Loop until done 패턴

```
ultracode

미결 PO 상태 일괄 업데이트 워크플로우:
대상: 미국오더_20260410-_클로드_정리_updated.xlsx → 미국오더-클로드정리 시트
Status가 비어있는 행을 모두 찾아서:
1. 분류 에이전트: 고객사별 그룹핑 (Skyworks / Micron / Qorvo)
2. 각 고객사 서브에이전트: 해당 PO의 Ship mode, Forwarder, Status 확인
3. 이상 항목(납기 초과, 미입력 등) 플래그 처리
4. 처리 완료된 항목 수 카운트

/goal 미결 항목 0개 달성 또는 수동 확인 필요 목록 출력까지
토큰 예산: 10k
```

## 5. 고객사 이메일 팩트 체크 — Deep Verification 패턴

```
quick workflow

아래 이메일 초안의 모든 수치/납기/사양 주장을 검증해줘:
[이메일 본문 붙여넣기]

1. 주장 추출 에이전트: 숫자, 날짜, 스펙 항목 리스트업
2. 항목별 검증 서브에이전트: PO 파일 / OOR 파일과 교차 대조
3. 불일치 항목 빨간 표시로 리포트 출력

자기 확증 편향 방지를 위해 작성자(나)의 판단 개입 없이 독립 검증
```

## 6. 영업 현황 근본원인 분석 — Root-cause 패턴

```
ultracode

5월 해외영업 실적 부진 원인 분석 워크플로우:
(Taiwan 목표 대비 71% 달성 원인 규명)

독립 가설 에이전트 3개 병렬 실행:
- 에이전트A: OOR 데이터 분석 (납기 지연 패턴)
- 에이전트B: 고객사별 오더 수량 추이 분석
- 에이전트C: 경쟁사/시장 요인 (웹 검색 활용)

각 에이전트 결과를 검증 에이전트가 교차 심사:
- 자기 확증 편향 차단: 각 에이전트는 다른 에이전트 결과 못 봄
- 검증 통과한 가설만 최종 리포트에 포함

출력: 김인섭 이사 보고용 원인분석 초안 (한국어)
```

## 7. 세션 룰 마이닝 — Memory 패턴

```
ultracode

지난 30개 세션 로그를 분석해서 CLAUDE.md 룰 자동 생성:
1. 로그 분석 에이전트들(병렬): 각 세션에서 내가 반복 수정한 패턴 추출
2. 클러스터링 에이전트: 유사 패턴 그룹핑, 중복 제거
3. 검증 에이전트(회의론자 페르소나): "이 룰이 실제 실수를 막았을까?" 역검증
4. 검증 통과 룰만 CLAUDE.md 추가 초안으로 출력

/goal 최소 5개 유효 룰 도출까지
```
