# Make.com + Claude (Anthropic API) KOSTAT 업무 적용 방안

> 이 문서는 `09. Make.com + Claude (Anthropic API) 자동화 구현 가이드`를 기반으로 KOSTAT 해외영업 업무에 Make.com을 도입할 때의 구체적인 적용 방안을 분석한다.

---

## 1. 현행 자동화 대비 Make.com 도입의 의미

| 항목 | 현행 (POP3 Watcher + Claude API) | Make.com 도입 시 |
|------|----------------------------------|----------------|
| 개발 방식 | Python 코드 직접 작성 | 모듈 드래그 + 설정 |
| 유지보수 | 코드 수정 필요 | UI 기반 수정 |
| 시나리오 가시성 | 로그 파일로만 확인 | 실시간 실행 내역 시각화 |
| 통합 확장성 | REST API 직접 구현 | 1,500+ 앱 모듈 (Slack, Notion, Gmail 등) |
| AI 연동 | Claude API 직접 호출 | HTTP 모듈로 Anthropic API 호출 |
| 스케줄링 | Task Scheduler 의존 | 내장 Schedule 모듈 |
| 장애 대응 | 로그 기반 수동 대응 | 자동 재시도 / 오류 처리 라우트 |
| 설치 필요 | Python 환경 + Task Scheduler | **설치 불필요 (클라우드 SaaS)** |

**결론**: 현행 POP3 Watcher는 잘 작동 중이므로 **대체보다는 보완** 관점에서 접근. Make.com은 설치 부담이 없고 즉시 시작 가능.

## 2. KOSTAT 적용 가능 시나리오 6선

### SC-1: 고객사 뉴스 모니터링 (난이도 ★☆☆)

```
Schedule (매일 오전 8:00, 평일만)
→ HTTP: Brave Search API — "Skyworks semiconductor latest news"
→ HTTP: Brave Search API — "Micron semiconductor latest news"
→ HTTP: Brave Search API — "Infineon technologies latest news"
→ Array Aggregator: 3개 검색 결과 병합 → 중복 제거
→ HTTP: Anthropic API (Claude) — "검색 결과를 요약, 고객사별 영향도 평가"
→ Telegram: 일일 브리핑 전송
```

**기대 효과**: 고객사 동향 수동 모니터링 시간 절감
**소요 시간**: 가입 후 30분 내 첫 실행 가능

### SC-2: PO 상태 알림 (난이도 ★★☆)

```
Schedule (매일 오전 9:00 / 오후 3:00)
→ Tools > Get Variable: PO Excel 파일 경로
→ HTTP: 사내 API 또는 공유 폴더에서 xlsx 데이터 조회
→ Tools > Parse JSON: ship_date +7일 필터
→ HTTP: Anthropic API (Claude) — "임박 PO 우선순위 요약"
→ Telegram: PO 알림 전송
```

**기대 효과**: 납기 임박 PO 누락 방지, remarks 누락 알림
**핵심**: Excel을 직접 읽는 대신 `kostat_excel_mcp.py`를 HTTP API로 감싸서 접근

### SC-3: OOR 주간 리포트 생성 (난이도 ★★☆)

```
Schedule (매주 금요일 오후 5:00)
→ HTTP: OOR 데이터 API 호출
→ Array Aggregator: 데이터 정리
→ HTTP: Anthropic API (Claude)
   - 프롬프트: "PO# 불일치, 날짜 지연, 수량 차이 분석. 전주 대비 변동. JSON 출력"
→ Text Parser: JSON 블록 추출
→ Notion: 주간 리포트 저장
→ Telegram: 요약 알림 전송
```

**기대 효과**: `X-ray` 모드의 정기 자동화 버전

### SC-4: 이메일 수신 → PO 자동 처리 (난이도 ★★★)

```
Gmail Trigger (새 이메일 감지)
→ Filter: 제목에 "PO" 또는 "Purchase Order" 포함
→ HTTP: 첨부파일(PDF) 다운로드
→ HTTP (PDF.co API): PDF → 텍스트 변환
→ HTTP: Anthropic API (Claude) — "PO 데이터 JSON으로 파싱"
→ Notion: PO 데이터 저장
→ Telegram: "PO#1234 업데이트 완료" 알림
```

**기대 효과**: POP3 Watcher를 Make.com으로 보완. IMAP/POP3 대신 Gmail Trigger 사용.
> **중요**: 현재 POP3 Watcher가 담당하는 메일 감시는 유지. Make.com은 Gmail 연동으로 추가 채널 확보.

### SC-5: 커미션 인보이스 월간 자동화 (난이도 ★★★)

```
Schedule (매월 1일 오전 9:00)
→ HTTP: 월간 실적 데이터 API 호출
→ HTTP: Anthropic API (Claude) — "고객사별 커미션 계산, 인보이스 포맷 JSON 출력"
→ JSON: 데이터 구조 검증
→ Tools > Compose: 인보이스 본문 생성
→ Notion: 정산 내역 기록
→ Telegram: 월간 커미션 요약 전송
```

**기대 효과**: 매월 반복 수작업 제거

### SC-6: Webhook 기반 PO 조회 비서 (난이도 ★★★)

```
Webhook (외부 POST 요청 수신)
→ Router: 의도 분류 (PO 현황 / OOR 현황 / 고객사 정보)
   ├── "PO 현황" → HTTP: PO 데이터 조회 → Claude 요약 → Webhook 응답
   ├── "OOR 현황" → HTTP: OOR 데이터 조회 → Claude 요약 → Webhook 응답
   └── "고객사 정보" → No… → Claude → Webhook 응답
```

**기대 효과**: Slack / Telegram Bot에서 `/po` 명령어로 조회 가능
> **Make.com Webhook**은 외부 앱(Slack, Telegram)에서 호출 가능한 URL을 생성한다.

## 3. 권장 도입 로드맵

| 단계 | 내용 | 시나리오 | 소요 시간 | Make.com ops/월 |
|------|------|----------|----------|----------------|
| 1단계 | Make.com 가입 + 기본 모듈 테스트 | SC-1 (뉴스) | 30분 | ~60회 (1일 1회 × 5일 × 4주) |
| 2단계 | Claude + Telegram 연동 | SC-3 (OOR) | 2시간 | ~20회 (주1회 × 4주) |
| 3단계 | HTTP API로 데이터 조회 | SC-2 (PO 알림) | 4시간 | ~160회 (1일 2회 × 20일) |
| 4단계 | Gmail Trigger + 자동 처리 | SC-4 (PO 자동) | 1주 | 변동 (수신량 기반) |
| 5단계 | Webhook + Router + 조건 분기 | SC-6 (PO 비서) | 1주 | ~100회 |

> **무료 티어(1,000 ops/월)로 1~3단계 충분히 커버 가능.** 4단계부터 유료 플랜 필요.

## 4. Make.com vs 현행 POP3 Watcher 비교

| 항목 | POP3 Watcher | Make.com |
|------|-------------|----------|
| 설치 | Python + Task Scheduler 필요 | **설치 불필요 (브라우저만)** |
| 실시간 메일 감시 | POP3 폴링 (5분 간격) | Gmail Trigger (거의 실시간) |
| 확장성 | 코드 수정 필요 | 모듈 추가로 확장 |
| AI 활용 | Claude API 직접 호출 | HTTP 모듈로 Claude 호출 |
| 실패 처리 | try/except 수동 구현 | 오류 처리 라우트 내장 |
| 로그/추적 | 로그 파일 | 실행 내역 대시보드 |
| 운영 비용 | PC 켜져 있어야 함 | 클라우드 (24/7) |
| 비용 | PC 전기세 | 무료 1,000 ops/월, 이후 유료 |
| 커스터마이징 | 완전 자유 | 모듈 기반 제약 있음 |

**결론**:
- **POP3 Watcher 유지** — Mailplug POP3 전용. 실시간 메일 감시.
- **Make.com 병행** — Gmail 연동, 정기 배치 분석, Webhook. 설치 부담 제로.
- **장기 검토** — Make.com이 안정적이면 POP3 Watcher를 Make.com으로 단계 이전.

## 5. Make.com × kostat_excel_mcp.py 연동 방안

Make.com은 Python MCP 서버를 직접 호출할 수 없다. 대신 **HTTP API로 한 번 감싸서** 연동한다.

### 현재: MCP Server (STDIO)
```
CMA MCP Server (kostat_excel_mcp.py)
  ↓ STDIO (Python subprocess)
Claude (Claude Code)
```

### Make.com 연동 시: MCP Server → HTTP API로 확장

kostat_excel_mcp.py에 HTTP 엔드포인트 추가:

```python
# 기존 @mcp.tool() 유지 + HTTP API 추가
from fastapi import FastAPI

app = FastAPI()

@app.get("/api/po/status")
async def query_po_status(customer: str = None, days: int = 30):
    """Make.com이 HTTP로 호출하는 엔드포인트"""
    df = pd.read_excel(PO_PATH)
    if customer:
        filtered = df[df['customer'].str.contains(customer, case=False)]
    else:
        filtered = df
    return filtered.to_dict(orient='records')
```

그러면 Make.com의 HTTP 모듈이 이 API를 호출:
```
URL: http://localhost:8000/api/po/status?customer=Skyworks&days=30
Method: GET
```

> MCP Server를 HTTP API로 확장하면 Claude Code / Make.com 둘 다 사용 가능.

**Make.com용 API 엔드포인트 리스트:**
| 엔드포인트 | 기능 | 활용 시나리오 |
|------------|------|-------------|
| `GET /api/po/status` | PO 상태 조회 | SC-2, SC-6 |
| `GET /api/oor/summary` | OOR 현황 요약 | SC-3, SC-6 |
| `POST /api/po/remarks` | PO Remarks 업데이트 | SC-4 |
| `POST /api/commission/calc` | 커미션 계산 | SC-5 |

## 6. Make.com 무료 티어 운영 전략

Make.com 무료 티어는 **1,000 operations/월**. 효율적으로 쓰는 법:

| 시나리오 | ops/회 | 월간 ops | 누적 |
|----------|--------|----------|------|
| SC-1: 고객사 뉴스 (1×/일 평일) | ~3 (Brave×3 + Agg. + Claude + Telegram) | 6 ops × 22일 = **132** | 132 |
| SC-3: OOR 리포트 (1×/주) | ~5 (HTTP + Agg. + Claude + Notion + Telegram) | 5 ops × 4주 = **20** | 152 |
| SC-2: PO 알림 (2×/일 평일) | ~4 (HTTP + Parse + Claude + Telegram) | 8 ops × 22일 = **176** | 328 |
| **합계** | | | **328 ops/월** |

> **무료 티어(1,000 ops) 안에서 1~3단계 충분히 운영 가능.** 유료는 4단계부터.

## 7. 한계 및 리스크

1. **Make.com MCP 미지원**: MCP 프로토콜을 직접 쓸 수 없어 HTTP 우회 필요
2. **Excel 직접 읽기 제한**: Make.com은 로컬 파일 직접 접근 불가 → API 서버 필요
3. **무료 티어 제한**: 1,000 ops/월, 1개 시나리오 활성화, 데이터 보관 1일
4. **한국어 지원**: Make.com UI가 영어 위주, 러닝커브 존재
5. **Mailplug POP3 미지원**: POP3 Watcher의 메일 감시는 Make.com으로 대체 불가 (POP3 모듈 없음)
6. **Claude 전용 모듈 부재**: OpenAI 모듈은 있지만 Claude는 HTTP로 직접 호출해야 함
