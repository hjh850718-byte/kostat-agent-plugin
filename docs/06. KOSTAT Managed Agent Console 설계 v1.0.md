# KOSTAT Managed Agent Console 설계 v1.0

> Claude Managed Agent (CMA) 기반 KOSTAT 해외영업 전담 에이전트 설계
> Console에서 Agent/Environment 생성 → Python SDK에서 sessions.create() 호출

```
┌──────────────────────────────────────────────┐
│  Console (https://platform.claude.com)        │
│                                              │
│  Agent: kostat-agent                         │
│  ├─ model: claude-opus-4-8                   │
│  ├─ system: [KOSTAT system prompt]           │
│  ├─ tools: agent_toolset_20260401            │
│  │          + custom (update_po, send_email) │
│  ├─ skills: xlsx, pdf                        │
│  └─ ❌ MCP 서버 불필요                        │
│                                              │
│  Environment: kostat-env                     │
│  └─ networking: unrestricted                 │
└──────────────────┬───────────────────────────┘
                   │ Agent ID + Environment ID
                   ▼
┌──────────────────────────────────────────────┐
│  run_kostat_agent.py (Python SDK)            │
│                                              │
│  1. files.upload(Excel) → file_id            │
│  2. sessions.create(                          │
│       resources=[{file_id, mount_path}])     │
│     → Container에 /workspace/data.xlsx 마운트 │
│  3. events.stream()                          │
│  4. events.send(user.message)                │
│  5. Event Loop:                              │
│     ├─ agent.message → 출력                  │
│     ├─ agent.tool_use(bash) → Excel 읽기      │
│     ├─ custom_tool_use → App 처리            │
│     └─ session.status_idle → 종료            │
└──────────────────────────────────────────────┘
                     │
          ┌──────────┴──────────┐
          ▼                     ▼
  Container 내 Agent      App (Python)
  ┌─────────────────┐   ┌─────────────────┐
  │ /workspace/      │   │ update_po()     │
  │  └─data.xlsx     │   │  → Excel 쓰기   │
  │ bash+openpyxl    │   │ send_email()    │
  │  → Excel 읽기    │   │  → Outlook 발송  │
  └─────────────────┘   └─────────────────┘
```

> **MCP 서버를 사용하지 않는 이유:** Excel 파일을 Session Resource로 직접 마운트하면
> Container 내부에서 bash+Python(openpyxl)로 읽을 수 있습니다. MCP 서버 호스팅이
> 필요 없어 아키텍처가 단순해집니다. 데이터 변경은 Custom tool로만 처리하여 안전성을 확보합니다.

## 핵심 업무

### 1. PO (Purchase Order) 관리
- PO PDF/Excel 파싱 → 고객사별 오더 현황 업데이트
- PO 번호, 고객사, 품목, 수량, 금액, 납기 추출 및 검증
- 중복 PO, 누락 PO 확인
- Excel 시트: 미국오더_업데이트.xlsx (US/ATP/MICRON 시트)
- **Custom tool 호출**: update_po(), lookup_po()

### 2. OOR (Order of Revenue) 분석
- 고객사별 매출 인식 현황 조회
- 월간/분기별 추이 분석 및 변동 원인 파악
- 이상 신호 감지 (목표대비 -10% 이상)
- **Custom tool 호출**: get_oor_data(), analyze_oor()

### 3. 커미션 인보이스
- RAW DATA 기반 요율 역산
- 고객사별(US/ATP/MICRON) 커미션 계산
- 인보이스 생성 및 검증
- **Custom tool 호출**: calc_commission(), generate_invoice()

### 4. 고객 이메일 관리 (MCP: kostat-email)
- Outlook 이메일 조회/검색/분류
- 고객사별(Skyworks/Qualcomm/Micron/Xview/Prent) 이메일 식별
- PO 관련 이메일 → Notion 로그 저장
- 납기/클레임/가격 논의 내용 구조화

### 5. 비즈니스 문서 번역
- 영문 → 한글 / 한글 → 영문 비즈니스 문서 번역
- 통계/무역 용어는 KOSTAT 공식 용어 사용
- 번역문은 원문 구조 유지, 전문 용어 일관성 확보

## 데이터 출처 우선순위

1. `/workspace/data.xlsx` (Session Resource로 마운트된 Excel) — 가장 신뢰度高
2. 첨부된 PDF/Excel 파일 — 1회성 데이터
3. 사용자 발화 — 최하위, 항상 데이터로 재확인

## 파일 규칙

- 날짜 형식: YYYY-MM-DD 통일
- 통화 표시: USD (기본), KRW (필요시) 명확히 구분
- PO 번호: 하이픈 포함 전체 표기 (예: PO-2401-001)
- 고객사명: 영문 공식명 (Skyworks Solutions, Qualcomm, Micron Technology)

## 5. Session 생성 및 실행 흐름 (SDK)

```python
# 1. Excel 파일 업로드
with open("data.xlsx", "rb") as f:
    uploaded = client.beta.files.upload(file=f)

# 2. Session 생성 + Excel 마운트
session = client.beta.sessions.create(
    agent=AGENT_ID,
    environment_id=ENVIRONMENT_ID,
    title="PO 업데이트 - Skyworks",
    resources=[{
        "type": "file",
        "file_id": uploaded.id,
        "mount_path": "/workspace/data.xlsx",
    }],
)

# 3. Stream 열기 (선 연결) + 메시지 전송
with client.beta.sessions.events.stream(session_id=session.id) as stream:
    client.beta.sessions.events.send(
        session_id=session.id,
        events=[{"type": "user.message", "content": [...]}],
    )

    # 4. 이벤트 루프
    for event in stream:
        match event.type:
            case "agent.message":          # Agent 텍스트 출력
            case "agent.tool_use":         # bash/openpyxl로 Excel 읽기 중
            case "agent.custom_tool_use":  # App에 처리 위임
            case "session.status_idle":    # 종료 or tool 대기
            case "session.status_terminated":  # 종료
```

## 6. Custom Tool 처리 흐름 (앱 담당)

```
Agent가 'send_email' 호출
  ↓
agent.custom_tool_use 이벤트 발생
  ↓
앱(파이썬)이 Outlook API로 실제 발송
  ↓
user.custom_tool_result 반환
  ↓
Agent가 결과 확인 후 다음 액션
```

이 구조의 핵심: **실제 데이터 변경/외부 발송은 앱에서 처리**.
Agent는 요청만 하고 결과를 받아서 사용자에게 설명합니다.

## 10. 제약사항 및 주의점

| 항목 | 내용 |
|------|------|
| **Agent는 1회 생성** | `agents.create()`는 SETUP에서만. 매 실행 금지 |
| **Model 변경 불가** | Session 생성 시 model 지정 불가. Agent 업데이트로만 |
| **Tool 변경 불가** | Session 중 tool 변경 필요 시 `sessions.update()` (idle 상태에서만) |
| **Archive는 영구** | Agent/Environment archive는 되돌릴 수 없음 |
| **Excel 파일 읽기 전용** | Container에 마운트된 파일은 수정 불가. 변경은 Custom tool로만 |
| **pip 설치 필요** | 최초 Session에서 `pip install openpyxl` 실행 필요 |
| **Session Output** | Agent가 `/mnt/session/outputs/`에 쓴 파일은 Session 종료 후 `files.list(scope_id=session.id)`로 다운로드 |
| **Stream 선 연결** | 이벤트 유실 방지를 위해 메시지 전송 전에 stream 먼저 열기 |
