### 2.2 Hook 설정 통합

Plugin의 `hooks/` 하위에 모든 Python 스크립트를 포함.  
설치 후 사용자가 `settings.json`에 hook 등록 필요 — 또는 Plugin 설치 시 자동 등록.

`hooks/hooks.json` (참조용 — Plugin이 자동 등록할 hook 설정):
```json
{
  "UserPromptSubmit": [
    "python {{PLUGIN_DIR}}/hooks/kostat-eod-detector.py"
  ],
  "PreToolUse": {
    "Write|Edit": [
      "python {{PLUGIN_DIR}}/hooks/kostat-gateguard.py",
      "python {{PLUGIN_DIR}}/hooks/kostat-autobackup.py"
    ]
  },
  "PostToolUse": {
    "Write|Edit": [
      "python {{PLUGIN_DIR}}/hooks/kostat-observe.py",
      "python {{PLUGIN_DIR}}/hooks/kostat-tracelogger.py"
    ]
  },
  "Stop": [
    "python {{PLUGIN_DIR}}/hooks/kostat-eod-stop.py"
  ]
}
```

**현재 Claude Code Plugin 시스템은 hook 자동 등록을 완벽히 지원하지 않을 수 있음.**  
→ 대안: `kostat-agent-setup` 스킬을 Plugin에 포함시켜, 사용자가 호출 시 settings.json에 hook 추가

## 3. 설치 방법 (설계)

### 방법 A: GitHub Marketplace (권장)

1. `kostat-agent-plugin`을 GitHub 리포지토리에 업로드
2. 아래 명령어로 설치:
   ```
   claude plugins install https://github.com/사용자/kostat-agent-plugin
   ```
3. 최초 실행 시 `kostat-agent-setup` 스킬이 자동 안내
4. 기술적 제약: 현재 Claude Code가 외부 리포지토리 Plugin 설치를 지원하는지 확인 필요

### 방법 B: 로컬 디렉토리 (확정)

```
git clone / 로컬에 복사
-> .claude/plugins/kostat-agent/ 에 직접 배치
-> settings.json에 "kostat-agent" Plugin 활성화:
    "enabledPlugins": {
      "kostat-agent@local": true
    }
```

### 방법 C: 마켓플레이스 등록 (장기)

- Claude Plugins 마켓플레이스에 등록
- 버전 관리, 업데이트 알림, 커뮤니티 기여 가능

## 5. 트레이드오프 분석

### 장점

| 항목 | 현재 (스킬 분산) | Plugin |
|------|-----------------|--------|
| 설치 | 수동 복사 2개 위치 | `plugins install` 한 번 |
| 버전 관리 | 없음 | package.json + Git tag |
| 이식성 | 낮음 (경로 의존) | 높음 (상대 경로) |
| 유지보수 | 파일 분산 추적 어려움 | 단일 디렉토리 |
| 공유 | 불가능 | GitHub / Marketplace |

### 단점 / 리스크

| 리스크 | 내용 | 대응 |
|--------|------|------|
| Hook 자동 등록 미지원 | Plugin이 settings.json hook을 자동 등록 못할 수 있음 | Setup 스킬에서 수동 등록 안내 |
| 경로 호환성 | 기존 스크립트가 절대 경로 하드코딩 | 상대 경로로 변경 필요 |
| Claude Code 업데이트 | Plugin 시스템 변경 시 호환성 깨질 수 있음 | 버전 고정, Release Note 추적 |
| 중복 관리 | Plugin + 기존 `.claude/skills/` 모두 남으면 충돌 | 마이그레이션 후 기존 스킬 삭제 |

## 7. 참고: Plugin SKILL.md 경로 변경

Plugin으로 전환 시 skill 파일 경로가 변경되므로,  
`settings.json`의 `enabledPlugins`에 Plugin 등록 후  
`.claude/skills/`의 기존 KOSTAT 스킬들을 제거해야 중복 등록을 방지합니다.

```
변경 전: .claude/skills/kostat-po-update/SKILL.md
변경 후: .claude/plugins/kostat-agent/skills/kostat-po-update/SKILL.md
```

Claude Code는 `skills/` 디렉토리와 `enabledPlugins` 모두에서 skill을 발견하면 중복 등록하므로,  
Plugin 활성화 후 기존 `.claude/skills/kostat-*` 디렉토리를 삭제해야 합니다.
