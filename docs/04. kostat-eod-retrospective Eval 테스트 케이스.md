## 테스트 실행 절차

1. 가상의 세션 로그(또는 실제 당일 대화)를 컨텍스트에 로드
2. `업무끝` 또는 `/kostat-eod-retrospective` 호출
3. 생성된 KPT 파일을 다음 판정 기준으로 평가
4. 결과를 `evals/eod-retro-eval-log.md`에 누적 기록

## Test Case 1: PO 업데이트 + OOR 분석 세션

| 항목 | 내용 |
|------|------|
| **시나리오** | Skyworks PO 3건(6000046945~47) PDF 파싱 → Excel 입력 완료. OOR Bring Forward 15건 검토 중 2건 PO# 불일치 발견. 커미션 인보이스 초안 생성했으나 검증 전. 중간에 출하 지연 건 고객사 문의 전화 받음. |

| **판정 기준** | ① K/P/T 각 2개 이상 항목 ② 구체적인 수치 포함 (PO 3건, OOR 15건 등) ③ "열심히 했다" 등 추상 표현 없음 ④ First Action 3~5개, 긴급도 표시 있음 ⑤ 날짜 형식 "2026.06.XX (X요일 저녁)" 정확 ⑥ 파일 저장됨 |

## Test Case 2: 커미션 인보이스 집중 세션

| 항목 | 내용 |
|------|------|
| **시나리오** | US/ATP/MICRON RAW DATA 읽어서 4월 커미션 인보이스 생성. 요율 역산 검증에서 ATP 시트 금액 $1,234 불일치 발견 → 수정 완료. SUMMARY 시트까지 검증 완료 → 최종 저장. 에디 커미션 파일 2개 생성. |
| **기대 결과** | Keep: 인보이스 교차 검증으로 오류 발견한 점. Problem: ATP 요율 불일치 원인 미파악(임시 수정만 함). Try: 불일치 원인을 다음 달까지 추적해서 프로세스 개선. |
| **판정 기준** | ① 작업 대상(커미션 인보이스)과 수치($1,234)가 구체적으로 기록 ② Problem이 단순 나열이 아니라 원인 추정 포함 ③ Try가 "다음에는 주의" 수준이 아닌 구체적 액션 ④ 내일 First Action에 "ATP 요율 불일치 원인 추적" 포함 |

## Test Case 3: 영업 실적 분석 + 대면 미팅 세션

| 시나리오 | 5월 해외영업 실적 분석 (Taiwan 71% 달성). OOR 데이터로 납기 지연 패턴 발견 (3건). 김인섭 이사 보고용 원인분석 초안 작성 완료. 오후에 Skyworks 미팅 참여 (툴링 신규 프로젝트 논의). |
|---|---|
| **기대 결과** | Taiwan 71% 수치, 납기 지연 3건, 미팅 내용이 모두 KPT에 반영. Problem에 납기 지연 패턴 분석의 한계 인식 포함. |
| **판정 기준** | ① "71%", "3건" 등 구체적 수치 기록 ② 미팅 내용이 Keep/Problem 중 하나에 반영 ③ "오늘도 수고했다" 등 빈말 금지 ④ First Action에 보고서 후속 조치 포함 ⑤ 파일 저장 경로 정확 |

## Test Case 4: 장애/이슈 발생 세션

| 시나리오 | PO 업데이트 중 Excel 파일 손상됨 (임시 파일만 남음). 30분 분량 작업 복구. 원인: 저장 중 PowerShell 강제 종료. 복구 후 2시간 추가 소요. OOR 분석은 미뤄짐. |
|---|---|
| **기대 결과** | Problem에 Excel 손상 사고와 원인 기록. Try에 "저장 전 자동 백업 Hook 활성화" 같은 근본 대책 포함. Keep에 침착하게 복구한 점. |
| **판정 기준** | ① Problem에 구체적 사건(Excel 손상)과 원인(PowerShell 강제 종료) 기록 ② Try가 "조심한다" 수준 금지 → "자동 백업 Hook 설정" 같은 **실행 가능한 조치** ③ 미뤄진 OOR 분석이 First Action에 포함 ④ 긴급도 🔴 표시 존재 |

## Test Case 5: Notion 저장 검증

| 시나리오 | EOD KPT 생성 직후. "Notion에도 저장해줘" 추가 요청. |
|---|---|
| **기대 결과** | 로컬 파일 + Notion 업무일지 하위 페이지 둘 다 생성. Notion 페이지 URL 반환. |
| **판정 기준** | ① 로컬 파일 `KPT_YYYY-MM-DD.md` 존재 확인 ② Notion 페이지 생성됨 (URL 반환) ③ 로컬과 Notion의 내용 일치 ④ Notion 페이지가 올바른 상위 페이지(오늘 업무일지) 아래에 위치 |
| **Notion MCP** | `API-post-search`로 오늘 업무일지 페이지 ID 조회 → `API-patch-block-children` 또는 `API-post-page`로 하위 페이지 생성 |
| **에지 케이스** | Notion MCP 미연결 시 → 로컬 저장만 하고 사용자에게 Notion 미연결 안내 |
